# Startup Validator Skill

Take a raw business idea from concept to launch-ready plan through a structured 7-phase pipeline. Each phase builds on the last. The user can run the full pipeline or jump to any individual phase.

## Input Requirements

- **Required:** A business idea (can be as rough as a single sentence)
- **Optional:** Target audience, domain expertise, budget constraints, timeline

If the user provides only a vague idea, ask ONE clarifying question before proceeding. Don't over-interrogate — start validating and refine as you go.

---

## Phase 1: Idea Validation

**Goal:** Determine if this idea is worth pursuing before investing time.

### Steps:
1. **Problem-Solution Fit Check**
   - What specific problem does this solve?
   - Who has this problem? How frequently? How painfully?
   - What do people currently do instead (existing alternatives)?
   - Why would someone switch to this solution?

2. **Quick Market Signal Scan**
   - Search for competitors and adjacent solutions
   - Check if people are actively searching for this (keyword signals)
   - Look for communities discussing this pain point (Reddit, forums, X)
   - Identify any failed attempts and why they failed

3. **Validation Scorecard**
   Rate 1-5 on each dimension:
   - Problem severity (is this a painkiller or vitamin?)
   - Market size (enough people have this problem?)
   - Timing (why now? what changed?)
   - Founder-market fit (does the founder have an unfair advantage?)
   - Competitive moat potential (can this be defensible?)

4. **Verdict:** GO / PIVOT / STOP with clear reasoning

### Output:
- Problem statement (1-2 sentences)
- Validation scorecard with ratings
- Top 3 risks identified
- GO/PIVOT/STOP recommendation with reasoning

## Phase 2: Niche Discovery

**Goal:** Find the most winnable beachhead market to start in.

### Steps:
1. **Market Segmentation**
   - Break the broad market into 5-10 specific niches
   - For each niche: size estimate, accessibility, willingness to pay
   - Identify which niches are underserved vs. saturated

2. **Beachhead Selection Criteria**
   - Can you reach these people directly? (channels exist)
   - Will they pay enough to sustain the business?
   - Can you dominate this niche before expanding?
   - Does winning here create a bridge to adjacent markets?

3. **Niche Ranking**
   - Rank top 3 niches by attractiveness
   - Recommend primary niche with reasoning
   - Map expansion path from niche → broader market

### Output:
- Niche map with 5-10 segments
- Top 3 ranked niches with scoring rationale
- Recommended beachhead market
- Expansion roadmap (niche → adjacent → broad)

## Phase 3: Customer Pain Point Research

**Goal:** Deeply understand the target customer's world, language, and pain.

### Steps:
1. **Customer Avatar**
   - Demographics and psychographics
   - Day-in-the-life narrative
   - Goals, frustrations, and aspirations
   - Where they spend time online and offline

2. **Pain Point Hierarchy**
   - Primary pain (the #1 thing keeping them up at night)
   - Secondary pains (related frustrations)
   - Latent pains (problems they don't realize they have)
   - Emotional drivers behind the functional pain

3. **Voice of Customer Research**
   - Search for how real people describe this problem (their exact words)
   - Find reviews of competitor products — what do people complain about?
   - Identify the "I wish..." and "Why can't I just..." statements
   - Capture the language patterns for use in marketing

4. **Jobs to Be Done**
   - Functional job: What task are they trying to accomplish?
   - Emotional job: How do they want to feel?
   - Social job: How do they want to be perceived?

### Output:
- Customer avatar (1 page)
- Ranked pain point hierarchy
- Voice of customer swipe file (10+ real quotes/patterns)
- JTBD statement

## Phase 4: Lean Business Plan

**Goal:** Create a one-page business plan that covers the essentials without the fluff.

### Steps:
1. **Business Model Canvas (Lean)**
   - Value proposition (1 sentence)
   - Customer segments (from Phase 2)
   - Channels (how you reach them)
   - Revenue streams (how you make money)
   - Cost structure (major cost categories)
   - Key metrics (what 3-5 numbers matter most)
   - Unfair advantage (what can't be easily copied)

2. **Unit Economics**
   - Customer acquisition cost (CAC) estimate
   - Lifetime value (LTV) estimate
   - LTV:CAC ratio target
   - Payback period estimate
   - Break-even analysis (how many customers/units)

3. **Risk Assessment**
   - Top 3 assumptions that must be true
   - How to test each assumption cheaply
   - Kill criteria (what would make you stop)

### Output:
- One-page lean business plan
- Unit economics model with assumptions stated
- Risk register with test plans
- 90-day milestone targets

## Phase 5: MVP Design

**Goal:** Design the smallest possible product that tests the core value proposition.

### Steps:
1. **Feature Prioritization**
   - List all possible features
   - Categorize: Must-have / Nice-to-have / Future
   - Apply the "what's the ONE thing that makes this valuable?" test
   - Cut ruthlessly — MVP should have 3-5 core features max

2. **MVP Type Selection**
   Choose the right MVP format:
   - Landing page + waitlist (testing demand)
   - Concierge MVP (manual delivery of the service)
   - Wizard of Oz (looks automated, human behind the curtain)
   - Single-feature product (one thing, done well)
   - Marketplace seed (supply-side or demand-side first)

3. **Technical Scope**
   - Recommended tech stack (bias toward speed and simplicity)
   - Build vs. buy decisions for each component
   - Timeline estimate (target: 2-4 weeks to launch)
   - Budget estimate for MVP build

4. **Success Criteria**
   - Define what "validation" looks like (specific metrics)
   - Minimum sample size for decisions
   - Timeline for the experiment
   - Decision framework: scale / iterate / pivot / kill

### Output:
- MVP feature list (must-haves only)
- MVP type recommendation with reasoning
- Technical architecture overview
- Build timeline and budget estimate
- Success metrics and decision criteria

## Phase 6: Customer Acquisition Plan

**Goal:** Get the first 100 customers without spending a fortune.

### Steps:
1. **Channel Identification**
   - Where does the target customer already hang out?
   - Which channels allow direct, personal outreach?
   - Rank channels by: cost, speed, scalability, founder-fit
   - Select 2-3 primary channels (don't spread thin)

2. **Pre-Launch Strategy**
   - Build an audience before the product exists
   - Content strategy (what to post, where, how often)
   - Waitlist / early access mechanics
   - Partnerships and cross-promotions

3. **Launch Playbook**
   - Week-by-week plan for first 30 days post-launch
   - Outreach templates (cold email, DM, community posts)
   - Launch day checklist
   - PR / product hunt / community launch tactics

4. **Growth Loops**
   - Identify potential viral loops or referral mechanics
   - Word-of-mouth accelerators
   - Retention hooks that drive organic growth

### Output:
- Channel ranking matrix
- 30-day launch playbook (week by week)
- 5 outreach templates ready to customize
- Growth loop diagram
- First 100 customers roadmap

---

## Phase 7: Monetization Strategy

**Goal:** Design a pricing and revenue model that maximizes value capture.

### Steps:
1. **Pricing Research**
   - Competitor pricing analysis
   - Willingness-to-pay signals from customer research
   - Value-based pricing calculation (what's the ROI for the customer?)
   - Price sensitivity considerations

2. **Revenue Model Design**
   - Select model type: subscription, one-time, usage-based, freemium, marketplace cut, etc.
   - Tier structure (if applicable)
   - Free vs. paid boundary (if freemium)
   - Upsell / cross-sell opportunities

3. **Pricing Strategy**
   - Launch pricing (introductory / beta pricing)
   - Target pricing (post-validation)
   - Pricing page design recommendations
   - Discount and promotion strategy

4. **Revenue Projections**
   - Month-by-month projections for first 12 months
   - Conservative / base / optimistic scenarios
   - Key assumptions and sensitivity analysis

### Output:
- Pricing model with tier breakdown
- Competitor pricing comparison table
- 12-month revenue projection (3 scenarios)
- Pricing page copy recommendations

## Consolidated Output

When running the full pipeline, deliver a **Startup Validation Report** that includes:

1. **Executive Summary** (1 page)
   - Idea overview
   - Validation verdict (GO/PIVOT/STOP)
   - Key findings
   - Recommended next steps

2. **Full Analysis** (Phases 1-7 outputs compiled)

3. **Action Plan**
   - Prioritized task list for the next 30 days
   - Key decisions that need to be made
   - Resources needed (time, money, skills)
   - Milestones and check-in points

### Delivery Format:
- Default: Markdown artifact for review, then .docx on request
- Include data tables, scorecards, and frameworks inline
- Use clear section headers for easy navigation

## Usage Notes

### Running Individual Phases
Users can request any single phase: "Just do the niche analysis" or "Help me design my MVP." When running a single phase, ask for any context from previous phases that's needed.

### Iterative Refinement
After delivering any phase, ask: "Want me to go deeper on any section, or move to the next phase?" The user controls the pace.

### Web Research
Use web search liberally during Phases 1-3 to ground analysis in real market data. Don't just theorize — find actual signals.

### Tone
Use the Builder/Technical voice. The user is building something — be direct, opinionated, and practical. Flag bad ideas early. Celebrate good ones. No hand-holding.
